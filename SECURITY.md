# Security Policy / Политика безопасности

## Supported Scope / Поддерживаемая область

Security reports are accepted for the current `main` branch of this template repository.

Сообщения по безопасности принимаются для текущей ветки `main` этого шаблонного репозитория.

## Reporting / Как сообщить

Do not publish secrets, tokens, private logs, credentials, or exploit details in public issues or pull requests.

Не публикуйте секреты, токены, приватные логи, учётные данные или детали эксплуатации в публичных issues и pull requests.

Use GitHub private vulnerability reporting or security advisories when available. If a private channel is not available, open a minimal public issue without sensitive details and ask the maintainer to arrange a private handoff.

Используйте GitHub private vulnerability reporting или security advisories, если они доступны. Если приватный канал недоступен, создайте минимальный публичный issue без чувствительных деталей и попросите maintainer согласовать приватную передачу информации.

## Secret Exposure / Утечка секретов

If a secret is exposed, revoke or rotate it first. In the report, include only redacted values, affected file paths, commit hashes, and reproduction steps.

Если секрет уже попал в открытый доступ, сначала отзовите или перевыпустите его. В сообщении указывайте только отредактированные значения, затронутые пути, хеши коммитов и шаги воспроизведения.
